# Instruções do Copilot para o Projeto Frontend

## Contexto Geral

Este é um projeto de aplicação de upload de arquivos com frontend em React/TypeScript e backend em Node.js. O frontend usa Vite, Tailwind CSS, React Router e Lucide React para ícones.

## Estrutura do Projeto

- `packages/frontend/`: Aplicação React com TypeScript
- `packages/backend/`: Servidor Node.js para API de arquivos
- Usa monorepo com npm workspaces

## Tecnologias Principais

- **Frontend**: React 19, TypeScript, Vite, Tailwind CSS, React Router 7, Lucide React
- **Backend**: Node.js, Express (presumido)
- **Estilo**: Tailwind CSS com tema dark
- **Estado**: React hooks (useState, useEffect)

## Convenções de Código

- Componentes funcionais com hooks
- Tipagem TypeScript rigorosa
- Estilos com Tailwind CSS (classes utilitárias)
- Estrutura semântica HTML (section, article, header)
- Tema dark consistente
- Funções utilitárias para merge de classes (clsx + twMerge)

## Padrões de UI/UX

- Design dark moderno
- Bordas arredondadas (rounded-xl, rounded-lg)
- Sombras suaves (shadow-lg, shadow-2xl)
- Transições suaves (transition-all duration-200)
- Hover effects com escala e elevação
- Espaçamentos generosos (gap-6, p-6)
- Cores: bg-gray-900 (fundo), bg-gray-800 (containers), text-white (texto principal)

## Comportamentos Esperados

- Sempre manter consistência com o tema dark
- Usar ícones apropriados do Lucide React
- Implementar hover states e transições
- Garantir responsividade
- Seguir convenções de acessibilidade (alt texts, titles, etc.)
- Manter código limpo e bem tipado

## Regras Específicas

- Não usar classes CSS customizadas; preferir Tailwind
- Sempre importar hooks necessários do React
- Usar fetch para chamadas de API
- Formatar datas em pt-BR quando aplicável
- Manter estrutura de pastas organizada
